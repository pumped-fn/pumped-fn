## VM-Aligned Direction

- Keep hot objects shape-stable: create fields in consistent order, avoid adding/removing properties after construction.
- Keep arrays packed: avoid sparse writes, delete, and out-of-bounds sentinel loops on classified dependency arrays.
- Avoid per-call closures and async trampolines on hot paths. Allocate middleware closures only when middleware exists.
- Prefer cached classified metadata (`WeakMap`) over rediscovering dependency shape per resolve.
- Benchmark full suite after each change; tiny benches have high RME, so judge by repeated geomean plus affected stable benches.

## Next Experiments

1. Resource context method binding: replace per-resource `owner.exec.bind(owner)`/friends with shape-stable wrappers or cached bound method set.
2. Extension chain build cache: precompose extension wrappers by extension list and target kind where possible.
3. Dependency result object shape: generate result properties in classified dependency order and avoid mixed late writes from async `.then` branches where possible.
4. Invalidation queue: replace `Array.shift()` with head index if long invalidation chains become a measured bottleneck.
